# **2025 새싹 해커톤 | AI 서비스 기능 명세서** 

1. **서비스 개요 (Service)**

| 서비스명 | **소리 비서 : Sori** |
| :---: | :---- |
| 서비스 한줄 소개 | **청각 장애인을 위한 실시간 대화 통역 및 환경 소리 시각화 비서** |
| 개발 목표 | 청각 장애인이 일상생활에서 겪는 **소리 정보의 부재(위험 상황, 대화 맥락)**를 AI 기술로 해결하여 안전하고 독립적인 삶을 지원함. |
| 타겟 사용자 | **청각 장애인 및 난청인** (특히, 비언어적 소리 인지와 대화 맥락 파악에 어려움을 겪는 사용자) |
| 기대 효과 | 1. **안전 사고 예방**: 위험 소리(경적, 사이렌 등)의 방향과 거리를 시각적으로 인지.<br>2. **소통 능력 향상**: 대화의 텍스트 변환뿐만 아니라 감정 상태까지 파악하여 원활한 소통 지원. |

2. **시스템 구성도 (Architecture)**  
* **Input**: 스마트폰 마이크 (Audio Stream)
* **Processing (AI Engine)**:
    *   **Sound Analysis**: 소리 감지 및 분류 (SED), 음원 위치 추정 (DOA)
    *   **Speech Analysis**: 음성 인식 (STT), 감정 분석 (Emotion Analysis)
* **Output (User Interface)**:
    *   **Radar View**: 소리의 위치 시각화
    *   **Safety List**: 위험도에 따른 색상 구분 및 알림
    *   **Chat View**: 대화 내용 및 감정 이모티콘 표시

3. **핵심 기능 명세 (Feature Specification)**

| 기능명 | 기능 설명 | 입력/ 출력 데이터 | 관련 기술 및 알고리즘 | 구현 여부 (O/X) |
| :---: | :--- | :---: | :---: | :---: |
| **환경 소리 감지**<br>(Safety Monitoring) | 주변 소리를 분석하여 위험/일상 소리로 분류하고 위험도(High/Medium/Low)를 표시 | Audio → Class, Urgency | Sound Event Detection (SED) | **O (UI)** |
| **음원 위치 시각화**<br>(Radar View) | 소리가 발생한 방향과 거리를 레이더 형태의 UI로 직관적으로 시각화 | Audio → Angle, Distance | Direction of Arrival (DOA) | **O (UI)** |
| **실시간 음성 인식**<br>(Voice Recognition) | 상대방의 음성을 실시간으로 텍스트로 변환하여 채팅 말풍선으로 표시 | Voice → Text | STT (Whisper 등) | **O (UI)** |
| **감정 분석**<br>(Emotion Analysis) | 대화 내용과 음성 톤을 분석하여 화자의 감정(긍정/부정/중립)을 아이콘으로 표시 | Voice/Text → Emotion | NLP / Audio Sentiment | **O (UI)** |
| **안전 알림**<br>(Safety Alert) | 위험도가 높은 소리(사이렌 등) 감지 시 붉은색 테두리 및 강조 표시로 즉각 알림 | Urgency → UI Color | - | **O (UI)** |

4. **주요 기능 흐름도 (Flow)**  
* **메인 화면 (Home)**: '환경 소리 모드'와 '음성 인식 모드' 중 선택
* **환경 소리 모드**:
    1.  주변 소리 수집 및 분석
    2.  **Radar View**에 소리의 방향(각도)과 거리(m) 표시
    3.  하단 리스트에 소리 종류와 위험도에 따른 색상(빨강/주황/초록) 표시
* **음성 인식 모드**:
    1.  상대방 음성 수집
    2.  실시간 텍스트 변환 및 채팅창 표시
    3.  분석된 감정(예: 긍정, 부정)을 이모티콘과 라벨로 말풍선 위에 표시

5. **향후 발전 방향 (Future Work)**  
* **AI 모델 연동**: 현재 UI 프로토타입에 실제 TFLite 모델(SED, STT) 탑재 및 연동
* **웨어러블 연동**: 스마트워치 진동을 통한 햅틱 피드백 구현
* **사용자 맞춤 설정**: 알림 받고 싶은 소리 커스텀 기능 추가
